## Прод = этот сервер

- Прод работает **здесь**: systemd-юнит `series-tracker` (uvicorn :5000),
  деплой из ветки `main` (см. [docs/deployment.md](docs/deployment.md)).
  Это не «неприкасаемый чужой прод» — это рабочая система пользователя на
  этой машине; правки выкатываются осознанно, через git + рестарт.
- **Развёртывание — только из git** (`git pull` ветки `main`), без `scp`
  отдельными файлами. Рабочие данные (`app.db`, `.venv`, `logs/`,
  `web/dist`) — вне git (`.gitignore`), при `git pull` сохраняются.
- БД (`app.db`) — реальные данные пользователя. Схема фиксирована
  Alembic-миграциями (0001→…). Перед миграцией/рискованной правкой — бэкап.

## Принципы (по ним система строилась, держим и дальше)

- **Никаких чёрных ящиков / молчаливых потерь функций.** Непонятый кусок
  разбирается до конца или выносится вопросом; ничего не теряется молча.
- **Правильность — собственными тестами и чек-листом сценариев.**
- **Чинить фундамент, а не заплатками.** Если что-то «фиксится» —
  значит, архитектура кривая; переделать так, чтобы фикс не требовался.
- Жёсткие констрейнты: дизайн/UX фронта неприкосновенны; протоколы
  внешних систем (qBittorrent API, трекеры, TMDB) — как есть.

## Архитектура (кратко; подробно — [docs/architecture.md](docs/architecture.md))

Один asyncio-процесс: «тупая» шина (`core/bus.py`, только
publish/subscribe + wildcards) и модули (`modules/*`), общающиеся
конвертами (`core/envelope.py`: event / command / query / reply).
Request/reply реализован в обвязке `core/module.py` ПОВЕРХ pub/sub —
шина о нём не знает. Gateway (`modules/gateway`) — FastAPI, единственная
HTTP/SSE-точка; событие шины → SSE по таблице `SSE_MAP`.

Правила модулей:
- модуль импортирует только `core/*`; импорт из чужого модуля запрещён;
- **БД — приватная память модуля** (Р-7): каждая таблица/колонка
  принадлежит ровно одному модулю; чужие данные — только query через
  шину; свой SQL — в `repository.py`; значимые изменения — событием;
- **«шина — сигнал, БД — истина»**: задача персистится до публикации
  события; при старте модуль добирает pending из БД (`on_start`);
- блокирующие операции — в executor'ах с таймаутами; группа логирования
  = имя модуля.

### Анатомия модуля
- `module.py` — класс `…Module(BaseModule)`; в `register()` — `self.handle(...)`.
  **Докстринг `module.py` — контракт модуля** (queries/commands/события,
  ресьюмабельность). Читать ПЕРВЫМ при работе с модулем.
- `repository.py` — весь SQL модуля.
- `on_start()` — reconcile незавершённых задач.
- профильные файлы (`torrents/qbt_client.py`, `sources/parsers.py`,
  `scan/planner.py`) — внешние протоколы и чистая логика.

Карта топиков шины — [contracts/bus_topics.md](contracts/bus_topics.md).
Сборка процесса (порядок старта, env) — `run.py`.

### Конвейер
`scan` (оркестратор + расписание) находит раздачи через `sources`
(kinozal/rutracker/anilibria/VK), применяет `rules`, регистрирует в
`torrents` (qBit) или ставит VK-загрузки в `downloads` (yt-dlp). Готовый
файл: `slicing` → `renaming` → `library`. `catalog` агрегирует вклады
статусов. `gateway` транслирует события в SSE и HTTP — в queries/commands
(бизнес-логики в gateway нет).

## Фронт

> ⚠️ **ДВА фронта — не перепутать.** Боевой UI отдаётся с корня `/`.

| Фронт | Исходники | Что отдаётся | Маршрут |
|---|---|---|---|
| **Новый (БОЕВОЙ)** | `web/src` (Vite + Vue 3 + TS + PrimeVue) | `web/dist` (сборка) | **`/`** |
| Старый (архив, rollback) | `legacy_frontend/` | как есть | `/legacy`, ассеты `/static` |

**Любая правка интерфейса — в `web/src/...`, затем `cd web && npm run build`**
(пересобрать `web/dist`; он в `.gitignore`, собирается на месте). Добавил
эндпоинт — перегенерь типы: `npm run gen:api`. Правки в `legacy_frontend/`
на боевой UI НЕ влияют. Визуал/DOM/классы — **неприкосновенны**; правки
JS-слоя данных согласуются с пользователем.

## Команды

| Задача | Команда |
|---|---|
| Весь набор тестов | `.venv/bin/python -m pytest -q` |
| Один тест | `… -k "имя_теста"` |
| Миграции БД | `.venv/bin/alembic upgrade head` |
| Запуск (dev) | `.venv/bin/uvicorn run:app --port 5000` (без `ST_QBIT_URL` модуль torrents не стартует) |
| Сборка фронта | `cd web && npm run build` |
| Браузерная доставка (astar/anilibria_tv) | `.venv/bin/playwright install firefox` |
| Рестарт прода | `sudo systemctl restart series-tracker` (или `./restart.sh`) |

qBittorrent — Docker, WebUI `:8080`; секреты — в `.env` (вне git, шаблон
`.env.example`). Origin репозитория публичный: реальные данные пользователя
и golden-фикстуры в git не коммитятся.

**Намеренно «красные» тесты — НЕ регрессии:** `*_match_production` / `*_diff`
и `test_sources` с реальными дампами требуют golden-фикстур, которых нет
(старой системы для сверки больше не существует). Интеграционные тесты с
живым qBit зависят от его состояния.

## Дисциплина

- **Коммит после каждой согласованной находки/решения** — отдельный
  логический коммит на русском, **без AI-атрибуции** (ни трейлеров, ни
  «сгенерировано»). Деплой — из git, ветка `main`, + рестарт.
- **Архитектурные/функциональные/UI-правки — после явного согласия.**
  Сначала разбор («как сейчас → почему → что предлагаю»), потом по «да» —
  правка. Чтение/поиск/тесты — свободно.
- Эмпирика важнее предположений: поведение системы утверждать только
  после прямой проверки (API/БД/логи/скриншот).
