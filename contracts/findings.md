# Журнал находок — этап 0 (2026-06-11)

Каждая находка требует решения «на месте» при разборе соответствующего модуля.

## 1. Девять мёртвых таблиц в прод-БД
В app.db 29 таблиц, в models.py — 20. Лишние: active_scans,
advanced_renaming_patterns, parser_rule_actions, quality_patterns,
quality_search_patterns, renaming_patterns, resolution_patterns,
resolution_search_patterns, season_patterns. Ни одна не упоминается в
коде (grep по *.py чисто). Похоже, легаси старых версий движка правил.
**Предложение:** в Alembic-схему новой системы не включать; миграция
очистки прод-БД — после переключения.

## 2. Дубль маршрута /directory-picker-test
routes/__init__.py:26 и routes/test.py:6 регистрируют один путь в двух
блюпринтах. Какой выигрывает — зависит от порядка регистрации.
**Предложение:** тестовые страницы в новую систему не переносить
(решение при ревизии gateway).

## 3. Два PUT /api/media-items/<id>/ignore
media.py:56 (int:item_id) и media.py:85 (string:unique_id) — две
версии одного действия. Фронт (seriesCompositionManager.js) использует
строковый unique_id. int-версия — вероятно, мёртвый предок.

## 4. SSE-шторм series_updated
За ~20 минут записи: 1728 событий series_updated (полный объект Series
каждый раз) на фоне активной загрузки — каждый тик прогресса
транслируется через пересчёт статусов. Подтверждает приоритет ревизии
статусной модели и перехода на дельты (принцип 2).

## 5. Версии qBittorrent: прод vs стенд
Стенд: v5.2.1 (Docker, свежая). Версию прода уточнить при этапе 2 —
если сильно старее, сверить совместимость используемых вызовов API v2.

## 6. python-json-logger отсутствует в requirements.txt
Найдено ещё при /init: logger.py импортирует pythonjsonlogger, пакета
нет в requirements.txt — на чистой установке падение на импорте.
Для новой кодовой базы — учесть в зависимостях с первого дня.
