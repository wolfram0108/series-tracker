# Агрегатор многосезонных серий по TMDB — план

> Статус: **ГОТОВО** (шаги 1–6, развёрнуто, 228 passed). Живой пруф —
> серия 7 «Безупречный мир»: сезоны нейминга [0,1] → сезон 0 исключён →
> TMDB-кэш `{0:3, 1:286}` → агрегат 286.
>
> Фича — учёт числа серий и счётчика «скачано» для **многосезонных** серий
> (режим «несколько сезонов»), где реальный набор сезонов на момент
> добавления неизвестен и узнаётся из нейминга.

## Проблема

`series_tmdb_mappings` хранит **один** `tmdb_season_number` + `total_episodes`
(число серий ОДНОГО сезона). Счётчик «скачано» при этом считает файлы по
**всем** сезонам. Для многосезонника (поле сезона пустое — `isSeasonless`)
это рассинхрон: total = один сезон, «скачано» = все сезоны → «80/26».

На момент добавления число серий многосезонника **принципиально неизвестно**
(сколько сезонов добавит пользователь — заранее не сказать), поэтому в этом
режиме счётчик пуст. Точный набор сезонов появляется только позже:

| Источник | Когда сезоны известны |
|---|---|
| Торрент | после скана + **переименования** (файлы разложены по `Season NN`; сезон — в `torrent_files.extracted_metadata.season`) |
| VK | после **первого скана / построения композиции** (`media_items.season`) |

## Решения (согласованы с пользователем)

1. Набор сезонов — из **нейминга** (реальные), не из ручного ввода.
2. Считаем только сезоны, что есть в TMDB; **сезон 0 (спешелы) исключаем**
   (обычно с TMDB не совпадают).
3. Пересчёт — **событийный** (скан / перестроение / усыновление / построение
   композиции), без таймеров. Реактивно (как дата последнего скана).
4. Хранение per-season счётчиков — внутренняя деталь: кэш числа серий по
   сезонам шоу в маппинге, `total_episodes` = агрегат (сумма по реальным
   сезонам ≠0).

## Механика

```
событие (скан / рейм / усыновление / композиция)
        │
        ▼
metadata.seasons.recompute {series_id}                  ← центральный обработчик
        │
        ├─ catalog.series.get → source_type, есть ли tmdb-маппинг
        ├─ собрать РЕАЛЬНЫЕ сезоны (≠0):
        │     торрент → torrents.seasons {series_id}   (distinct extracted_metadata.season)
        │     VK      → scan.seasons {series_id}        (distinct media_items.season)
        ├─ TMDB-счётчики по сезонам:
        │     кэш season_episode_counts в маппинге;
        │     если появился сезон не из кэша → metadata.details(tmdb_id), обновить кэш
        ├─ total = Σ episode_count(реальные ∩ TMDB, ≠0)
        └─ сохранить total → publish series.updated → SSE (карточка сама обновится)
```

Одиночный режим (`season="s01"`) не меняется — там один сезон, как сейчас.

## Изменения по модулям

| Модуль | Изменение |
|---|---|
| **БД (Alembic 0005)** | `series_tmdb_mappings.season_episode_counts` (TEXT/JSON `{ "<season>": episode_count }`, кэш TMDB). `total_episodes` → агрегат |
| **metadata** | ядро `metadata.seasons.recompute {series_id}`; кэш per-season; агрегат; событие |
| **torrents** | query `torrents.seasons {series_id}`; дёргать recompute после `_complete` (рейм завершён) |
| **scan** | query `scan.seasons {series_id}`; дёргать recompute после `_scan_vk` / композиции |
| **downloads** | дёргать recompute после усыновления |
| **gateway** | карточка: total из агрегата; счётчик «скачано» — по сезонам ≠0 |

## Детали

- **Кэш TMDB** — сеть только на новый сезон; сам пересчёт (сумма) локальный,
  дёшев, идёт на каждом триггере.
- **Счётчик «скачано»** — файлы сезонов ≠0 (спешелы мимо) → «скачано ≤ всего».
- **Реактивность** — `recompute` публикует `series.updated`, карточка мёржит
  без F5 (тот же путь, что и дата последнего скана).
- **Включение** — только для seasonless + назначенного TMDB; иначе поведение
  прежнее.

## Порядок реализации (по коммитам)

1. **Миграция** `0005` — кэш-колонка. *(этот шаг)*
2. **metadata** — ядро recompute + кэш TMDB.
3. **queries** `torrents.seasons` / `scan.seasons`.
4. **триггеры** recompute (torrents / scan / downloads).
5. **карточка/счётчик** — агрегат + сезоны ≠0.
6. **тесты** — агрегатор, кэш (без сети при повторе), реактивность, триггеры.

Каждый шаг — отдельный коммит с прогоном тестов.
