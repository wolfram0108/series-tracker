## Релиз

Модульно-шинная система **выпущена и работает на проде** (этот сервер).
- `main` = новая система; тег **`v2.0.0`** + GitHub Release.
- Старая монолитно-агентная версия — тег **`legacy/v1-final`** (откат);
  старый фронт — `legacy_frontend/` (отдаётся по `/legacy`).
- Прод: systemd-юнит `series-tracker` (uvicorn :5000), деплой из `main`
  по [deployment.md](deployment.md).
- Тесты бэкенда зелёные (кроме намеренно красных `*_match_production` /
  `*_diff` / `test_sources` — нужны golden/прод-фикстуры; старой системы
  для сверки больше нет).

**Бэклог (не блокеры):** сквозной astar/anilibria_tv на реальной раздаче.

## Что починено в v2 (журнал ключевых находок)

Поведенческая матрица (полностью — [contracts/series_situations.md](../contracts/series_situations.md))
пройдена: торрент 62 ✓ / 5 ◐, VK 46 ✓ / 6 ◐, 0 нерешённых.

Торрент-конвейер:
- **R2** — re-add раздачи после удаления из qB не запускал конвейер.
- **R1/R3/P10** — скан не сверял файлы; исчезнувшая раздача оставляла фантом
  `is_active`; застрявшая (пауза 0%, без задачи) не перезапускалась.
- **Д1** — счётчик «скачано» не уезжал в SSE.
- **Д2** — удаление серии не прерывало in-flight yt-dlp/ffmpeg/перемещение и
  не сносило полуфайлы (принцип: удаление НИКОГДА не трогает готовые файлы).
- **F7** — при недоступном NAS `fs.verify` помечал всё ложно missing.
- **C9** — конвейерное переименование без per-series лока (гонка с reprocess,
  корень кривых имён `.mkv.mkv.mkv`).

VK:
- **R4** — обычный скан не сверял диск VK (`downloads.fs.sync`).
- **VF6** — `_remove_partial` не чистил реальные артефакты yt-dlp.
- **VP7/VL2/VL5** — `run_ffmpeg` писал неатомарно; битый кусок усыновлялся как
  готовый → теперь tmp→`os.replace`.
- **VF2** — обычный скан не сверял нарезанных детей.
- **VU11** — `ignored_seasons` писался, но не применялся в плане → починен.

TMDB: агрегирование числа серий по нескольким сезонам (многосезонник).

Остаточные ◐ (приемлемые): VU9 (приоритет не заменяет уже скачанное),
VP3 (живой репро yt-dlp «96% без файла» — по запросу), VP5/VS4 (рассинхрон
voiceover — имя лагает 1 цикл, перекачки нет), VC5 (save_path во время
загрузки).
